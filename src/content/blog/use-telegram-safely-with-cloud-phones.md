Telegram 是很多团队最常用的沟通工具。负责人可以在群里发送任务说明，运营可以回复处理进度，技术同事也可以快速确认某个账号或设备状态。

当团队使用 QCCBot 云手机时，Telegram 也常常成为入口：成员在聊天中收到 QCCBot 链接，打开后进入云手机列表，再进入对应的远程 Android 设备处理任务。

这个流程很方便，但也带来一个容易被忽视的问题：账号信息应该留在设备会话里，而不是被随手复制到聊天群。

## 两个不同的使用环境

在这个场景里，其实有两个不同环境。

第一个环境，是外部 Telegram 聊天。它负责沟通、交接和发送入口链接。

第二个环境，是 QCCBot 云手机里的远程 Android 设备。这个设备里可能安装了 Telegram，也可能已经登录了业务账号。

团队需要区分这两件事。外部 Telegram 群可以用来说明任务状态，但真正的账号操作应该在云手机会话中完成。

## 哪些信息可以放在群里

适合放在 Telegram 群里的，是不会暴露账号隐私的协作信息。

例如：

- QCCBot 登录入口；
- 云手机 ID；
- 设备是否运行；
- 当前任务是否完成；
- 哪台设备需要人工处理；
- 哪个项目由谁继续跟进。

这些信息能帮助团队快速协作，也不会直接暴露账号内容。

## 哪些信息应该留在云手机里

涉及账号安全和用户隐私的信息，应该留在 QCCBot 云手机会话里处理。

包括：

- Telegram 登录验证码；
- 账号密码和恢复信息；
- 私聊内容；
- 客户资料；
- 手机号、邮箱和邀请链接；
- 含敏感信息的截图；
- 账号警告或验证页面。

如果同事需要查看具体页面，最好让他进入对应云手机，而不是把截图发到群里。这样可以避免敏感信息在聊天记录中长期留存。

## 更好的任务交接示例

不清楚的交接通常会制造更多问题。例如只说“验证码发我一下”或“这个页面怎么处理”，其他人很难判断上下文，也容易把敏感内容发错地方。

更好的交接可以这样写：

“云手机 ID 2059... 当前停在 Telegram 登录邮箱页面。我会在云手机内完成登录，群里只同步处理进度。”

或者：

“客服云手机里有一条 Telegram 私聊需要人工确认，请进入对应设备查看，不需要在群里转发截图。”

这种写法既能让团队知道任务进度，又不会把账号信息从设备会话里带出来。

## 团队可以建立一条简单规则

可以把规则定得很简单：

**Telegram 群里同步状态，QCCBot 云手机里处理账号。**

如果一条消息只是说明“哪台设备、什么状态、谁来处理”，可以发在群里。

如果内容包含验证码、私聊、客户资料或账号恢复信息，就应该留在云手机里。

这条规则不复杂，但能减少很多不必要的账号风险。

## QCCBot 如何帮助团队更清楚地协作

QCCBot 让团队可以通过浏览器进入不同云手机，把不同账号、不同项目、不同 App 环境分开。运营人员不需要把所有账号都塞进一台实体手机，也不需要通过大量截图来交接任务。

当每台云手机都有明确用途时，团队只需要在 Telegram 中说明设备 ID 和任务状态，相关人员就能进入正确环境继续处理。
